Splunk SPL for exporting samples

Run these searches in Splunk Web, then click Export → CSV to save the results. Drop the saved files onto Paydirt above. Start with Discover sourcetypes to see what's available - adjust the index and sourcetype filter on the first line for your environment - then plug a chosen index and sourcetype into the other two searches.

Discover sourcetypes last 30 days 
| tstats count where index=* AND sourcetype={sourcetype(s)} earliest=-30d BY sourcetype, index, source
| stats values(index) as indexes, values(source) as sources, sum(count) as event_count by sourcetype
| sort -event_count
Field-value samples last 7 days 
index={index} sourcetype="{sourcetype}" earliest=-7d@d
| fieldsummary maxvals=5
| search field!="_*" AND field!="date_*" AND field!="linecount"
  AND field!="punct" AND field!="timestartpos" AND field!="timeendpos"
  AND field!="splunk_server_group"
Log samples last 1 day 
index={index} sourcetype="{sourcetype}" earliest=-1d@d
| dedup punct | head 20

Results

Review required before sending. Paydirt cannot detect bare personal names, free-text content, or organization-internal terms with no distinguishing pattern. Common blind spots: assigned_to / assignee / requester values, comment / description / notes fields, person-named assets, internal project or customer code names, branch and team names, and stack traces echoing user input. For structured fields, add an @json,<field_name>,<replacement> rule to your config and re-run; for narrative text, manual review is the only control.
